# Service 360 Subsystem

**Registration name:** `"S360"`
**Default enabled:** not in default `config.Subsystems`
**Files:** `Subsytems/S360/`

## Overview

Integrates CSChat with Service 360 (S360), Microsoft's internal service health and accountability platform. Surfaces active action items, KPI scores, and burndown trend slopes for registered services.

## S360Client

**File:** `Subsytems/S360/S360Client.cs`

`[IsConfigurable("S360")]`
`[DependsOn("Kusto")]`

Implements `ISubsystem`. On enable: adds the S360 command group. On disable: removes it.

### Built-in Kusto Config

S360 uses a fixed cluster endpoint (`https://s360prodro.kusto.windows.net`, database `service360db`, 30-second timeout) rather than requiring a user-configured `KustoConfig` entry. A transient `KustoConfig` is constructed at query time.

### KQL Queries

`BuildActionItemsKql(serviceIds)` queries `S360_Kpi_Metadata`, `GetActiveActionItems()`, `S360_ServiceDetails_RT`, and related tables. Results include KPI type, created-by, display name, decoded HTML description, and service details.

`S360Insights` computes burndown slopes and composite scores from wave snapshot data to identify services with worsening health trends.

## S360Config

**File:** `Subsytems/S360/S360Config.cs`

`S360Profile` — a `[UserManaged("S360 Profile", ...)]` type for configuring which services to monitor:

| Field | Required | Default | Description |
|-------|----------|---------|-------------|
| `Name` | yes (key) | — | Profile name |
| `ServiceIds` | yes | — | List of service GUIDs to track |
| `FreshDays` | no | 7 | Age threshold (days) for a "fresh" change |
| `DueSoonDays` | no | 7 | Days-until-due threshold for "due soon" |
| `W_RecentChange` | no | 1.0 | Weight: recent change |
| `W_Unassigned` | no | 1.0 | Weight: unassigned items |
| `W_DueSoon` | no | 2.0 | Weight: due soon |
| `W_SlaAtRisk` | no | 2.2 | Weight: SLA at risk |
| `W_MissingEta` | no | 1.6 | Weight: missing ETA |
| `W_ManyEtaChanges` | no | 1.2 | Weight: many ETA changes |
| `W_Delegated` | no | 0.6 | Weight: delegated items |
| `W_OffTrackWave` | no | 2.5 | Weight: off-track wave (end date passed, remaining > 0) |
| `W_BurndownNegative` | no | 1.8 | Weight: burndown non-negative/no-progress trend |
| `W_BurndownMinPoints` | no | 3 | Minimum points required to trust burndown slope |
| `OffTrackGraceDays` | no | 2 | Grace days after end date before flagging off-track |

## S360Insights

**File:** `Subsytems/S360/S360Insights.cs`

Computes health metrics:
- **WaveSnapshot** — a point-in-time count of action items by severity
- **Burndown slope** — linear regression over wave snapshots to detect improving vs. worsening trends
- Composite scoring combining slope, absolute count, and recency

## S360Tools

**File:** `Subsytems/S360/S360Tools.cs`

Registers `ITool` implementations for S360 data (list action items, get service score, query burndown) so the Planner can invoke them.

## Commands

**File:** `Subsytems/S360/S360Commands.cs`

Registers under the `S360` command group:

| Command | Description |
|---------|-------------|
| `S360 > action items` | List active action items for configured services |
| `S360 > scores` | Show composite health scores |
| `S360 > burndown` | Show burndown trend slopes |
| `S360 > profile` | Edit the S360Profile (service IDs to track) |

## Dependencies

Requires `Kusto` to be enabled. `SubsystemManager` auto-enables Kusto when S360 is enabled.
